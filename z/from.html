HTML Forms Learning Project

A complete, beginner-friendly reference for every HTML form element, input type, and attribute.

1. Basic Text-Based Inputs

Text-based inputs collect single-line string data. Each type triggers a different on-screen keyboard on mobile and different built-in validation.

required, minlength=2, maxlength=50, autofocus on page load
required, minlength=8 -- characters are hidden as dots
browser auto-validates format like name@domain.com
pattern enforces format 123-456-7890
must start with a valid scheme, e.g. https://
connected to a datalist below for suggestions
readonly: value is submitted, but user cannot edit it
disabled: value is NOT submitted with the form at all

9. Accessibility Examples

Accessible forms help screen reader users and keyboard-only users navigate easily. Always pair inputs with a <label>, and use fieldset/legend to group related fields.

aria-label provides a label for screen readers when no visible <label> exists
tabindex explicitly sets the keyboard "Tab" order between fields
10. Buttons
submits using the form's action="https://example.com/submit-form"
resets all fields to their initial/default values
type="button" never submits or resets a form by itself
clicking the image submits the form (also sends click x,y coordinates)
formaction/formmethod/formenctype/formtarget override the form's own attributes just for this button; formnovalidate skips validation for this submit only
11. Notes on Form-Level Attributes

This entire <form> uses method="post", enctype="multipart/form-data" (required because it contains file inputs), autocomplete="on", and accept-charset="UTF-8". To disable ALL built-in browser validation for the whole form at once, you would add the novalidate attribute directly on the <form> tag itself (not added here, so that validation examples above remain active).
